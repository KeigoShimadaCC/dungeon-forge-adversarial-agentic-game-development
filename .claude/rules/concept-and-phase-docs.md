# Concept And Phase Rules

- Treat `concept-and-ideas/` as product intent; do not rewrite it unless the user asks.
- Treat `PHASE-00A` as the global invariant contract.
- Before implementing a phase, read `PROGRESS.MD`, that phase file, and `PHASE-00A`.
- When editing phase plans or starting phase work, update `PROGRESS.MD` checklist and validation log as appropriate.
- Keep phase docs concise implementation briefs, not full source-code dumps.
- Preserve the 10-section phase format unless the user asks to change it.
- Do not promote roadmap features into MVP phases.
- If concept and phase docs conflict, stop and report the conflict.
- When adding phase docs, use `PHASE-XXA-SHORT-SLUG.md`.
