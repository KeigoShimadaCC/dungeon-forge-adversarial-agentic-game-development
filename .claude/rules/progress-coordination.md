# Progress Coordination Rules

- Read `PROGRESS.MD` before phase-scoped edits; update it during and after work.
- `PROGRESS.MD` is live coordination only—not product intent. Design truth stays in `concept-and-ideas/` and `phase-plans/`.
- **Task queue:** agents add granular tasks (`[ ]` open, `[~]` in progress, `[x]` done, `[-]` skipped, `[!]` blocked).
- **Phase checklist:** tick deliverables and acceptance only when verified against the repo, not on intent alone.
- **Future backlog:** record out-of-scope ideas with suggested phase; do not implement without user or phase authorization.
- **Validation log:** append commands, results, commits, PRs, and blockers; do not delete history casually.
- **Phase archive:** completed phases are read-only summaries; rotate per `PROGRESS.MD` when starting a new phase.
- Keep Active Phase table aligned with the real branch, worktree, and `phase-plans/PHASE-XX...md` file in use.
